Note on "Clamped"

So we now know TypedArray has specific size each element can hold, but what if we assigned value of more than its size? What happens if we put too much batter in each muffin tin? Not clamped array takes care of this situation by only talking over flow. So if you try to assign 280 into an element in Unit8Array, it will only assign 24 (280 - 256 = 24).

"Clamped" array takes care of this situation differently. Let's think about a case when we changed value of red. If the original color already had value of 255 and we added 30, it will try to assign 285 in our image data array. If this array was using Unit8Array then it will only take over flow which is 29. We'll end up in much dark red even though we wanted to brighten it. In color value, lowest is always 0 and highest is always 255. Clamped Array is designed to take care of there situation by alway choosing clossed value to the size.
